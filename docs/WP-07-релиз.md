# WP-07: релизная сборка и runbook

## Как собрать

Все команды — из junction-пути `h:\.pet\potok\app` (кириллица в оригинальном
пути ломает Dart-тулинг):

```powershell
$env:Path += ';C:\dev\flutter\bin'
Set-Location h:\.pet\potok\app

flutter analyze          # 0 issues
powershell -NoProfile -ExecutionPolicy Bypass `
  -File .\tool\prepare_default_asr_model.ps1
flutter test             # 155 тестов зелёные, 1 host-only ASR smoke пропущен
flutter build windows --release
flutter build apk --release
flutter build apk --release --split-per-abi
powershell -NoProfile -ExecutionPolicy Bypass `
  -File .\tool\generate_release_metadata.ps1
```

## Артефакты

| Платформа | Путь |
|---|---|
| Windows | `app\build\windows\x64\runner\Release\potok.exe` (+ DLL и `data\` рядом; для распространения нужен весь каталог `Release`) |
| Android | `app\build\app\outputs\flutter-apk\app-release.apk` |
| Android, меньше | `app-arm64-v8a-release.apk` для большинства современных устройств; рядом также `armeabi-v7a` и `x86_64` |
| SBOM/hashes | `app\build\release-metadata\potok.cdx.json`, `SHA256SUMS.txt` |

Последняя проверка после WP-10: Windows release собран за 29 с, весь каталог
занимает 157,1 MiB. Universal Android APK собран за 39,4 с и занимает
199,5 MB; split APK: arm64-v8a 107,4 MB, armeabi-v7a 97,8 MB, x86_64
112,4 MB. Рост относительно WP-09 — встроенная RU/EN ASR-модель (~98,8 MiB).
Windows распространяется всем каталогом `Release`, а не одним `potok.exe`.

## Windows UX (WP-07)

- Горячие клавиши: `Ctrl+N` текст, `Ctrl+Shift+N` аудио, `Ctrl+K` поиск,
  `Ctrl+S` немедленный durable flush документа, `Delete` — заметка в корзину
  (только вне текстовых полей, с Undo), `Esc` — закрыть detail / снять
  выделение.
- Настройки → «Система» (только Windows):
  - «Сворачивать в трей при закрытии окна» — по умолчанию выключено; при
    включении close прячет окно, выход через меню трея («Выход»).
  - «Глобальная горячая клавиша Ctrl+Alt+N» — по умолчанию выключено;
    конфликт регистрации честно показывается в настройках, приложение
    работает дальше.
- Обе настройки хранятся в `app_meta` (SettingsService, не синхронизируются).

### Зависимости Windows-shell

`window_manager 0.5.2`, `tray_manager 0.5.3` и `hotkey_manager 0.2.3`
зафиксированы в `pubspec.yaml`/`pubspec.lock`; все три — MIT. Они нужны
только для управления окном, tray и регистрации системного hotkey.
Новых сетевых вызовов, фоновых служб или permissions они не добавляют.
На Android shell-интеграция не активируется; вызовы нативных плагинов
сериализованы, ошибка регистрации hotkey не роняет приложение.

## Cold start (замер 2026-07-18)

Метод: `Start-Process potok.exe` + опрос `MainWindowHandle` каждые 25 мс
(release-сборка, Windows 11 x64), три последовательных запуска:
**352 / 188 / 188 мс**. Первый запуск укладывается в **0,36 с** от старта
процесса до появления окна — в пределах цели ≤2–3 с (ТЗ 0.13).
Инициализация в `main()`
минимальна: до `runApp` выполняется только
`windowManager.ensureInitialized()` (без обращений к диску); БД и сервисы
поднимаются асинхронно после первого кадра. Smoke: процесс жив после 8 с
работы, завершается корректно.

## Что не сделано (осознанно)

- **Подпись**: Windows-сборка не упакована в MSIX и не подписана;
  release APK подписан debug-ключом (для Google Play нужен собственный
  keystore и `signingConfig` в `android/app/build.gradle.kts`).
- **Минимальные версии ОС**: release APK заявляет Android minSdk 24
  (Android 7.0) и targetSdk 36; физическая device matrix ещё не пройдена. Windows
  проверен на Windows 11 x64, а минимум пока не закреплён пакетом/installer.
- **Device smoke** на физических Android-устройствах не выполнялся
  (собран только APK).
- Выбор Windows-микрофона, передача device ID и WAV-контейнер покрыты тестами,
  но реальный capture через конкретные драйверы/гарнитуры требует ручного smoke.
- Offline ASR работает из коробки для новых WAV PCM16/16 kHz записей на
  встроенной sherpa-onnx Whisper tiny RU/EN-модели; пользователь может выбрать
  другую совместимую модель. Старые AAC/M4A пока не декодируются для ASR;
  воспроизведение и хранение таких записей не затронуты.
- **Шифрование отсутствует by design** (ТЗ 0.12): данные хранятся локально
  без шифрования, это отражается в privacy notice.
- **Ограничение hotkey**: если системе не удаётся зарегистрировать
  `Ctrl+Alt+N`, показывается сообщение о конфликте; «тихий» перехват
  сочетания другим приложением без ошибки регистрации не детектируется.
- Tray и глобальный hotkey проверяются вручную (в автотестах плагины
  не эмулируются).
- Android release собирается, но Flutter 3.44.6 предупреждает о
  будущей несовместимости `quill_native_bridge_android` с Built-in Kotlin;
  перед следующим обновлением Flutter нужно повторить release-build и
  обновить `flutter_quill`/транзитивный плагин, когда он добавит поддержку.
- App-level `lintVitalRelease` зелёный. Полный Gradle `lintRelease` падает в
  собственном lint-таске `just_audio 0.10.6` (library minSdk 16, 137 `NewApi`
  findings); для этого не добавлен маскирующий baseline.

## Privacy-чек (0.10.2)

`grep debugPrint/print` по `lib/`: все диагностические сообщения печатают
только `runtimeType` ошибки, без содержимого заметок, путей пользователя,
уровней микрофона и другого приватного контента.

Release APK не имеет `INTERNET` и `ACCESS_NETWORK_STATE`; manifest содержит только
`RECORD_AUDIO`, notification и microphone foreground-service permissions (плюс
автоматическое internal permission AndroidX для non-exported receiver).

### ASR smoke вне `flutter_tester`

Windows Flutter test host может заранее загрузить несовместимый ONNX Runtime.
Поэтому фактическая загрузка модели проверяется отдельным процессом:

```powershell
dart run .\tool\asr_model_smoke.dart `
  .\assets\models\default `
  "$env:LOCALAPPDATA\Pub\Cache\hosted\pub.dev\sherpa_onnx_windows-1.13.4\windows"
```

Проверка 2026-07-18: sherpa-onnx загрузил модель и обработал синтетический WAV.
