# ADR-010 — удаление продуктовых сессий и локальные названия заметок

Дата: 2026-07-18  
Статус: принято владельцем продукта

## Контекст

Сущность Session была реализована как Should-функция ТЗ, но признана лишней.
Одновременно существующая nullable-колонка `Note.title` не используется, хотя
карточкам требуется короткое название по содержанию.

## Решение

- удалить Session из domain/application/presentation и schema v3;
- миграция удаляет session link/context, не удаляя заметки или media;
- удалить FR-SES и связанные пункты из актуального ТЗ;
- использовать отдельное `Note.title`;
- дефолтное название создаёт детерминированный локальный extractive generator
  из первой содержательной фразы текста/расшифровки;
- пользователь может изменить название; непустое ручное название не заменяется
  автоматикой;
- не добавлять вторую нейросеть только ради заголовка: это ухудшит размер,
  cold start и сложность лицензирования непропорционально задаче.

## Последствия

- backup schema v2 требует миграции при restore;
- экспорт больше не содержит session context;
- термин `session` допустим только для технических API (например,
  `audio_session`), не как доменная сущность;
- эвристический заголовок не является абстрактивным AI-summary, но полностью
  offline, мгновенный и предсказуемый.
