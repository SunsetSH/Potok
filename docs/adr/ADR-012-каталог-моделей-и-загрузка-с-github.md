# ADR-012 — каталог ASR-моделей и загрузка с GitHub Releases

Дата: 2026-07-19  
Статус: принято

## Контекст

Bundled Whisper tiny ([[ADR-009]]) давал рабочий старт «из коробки», но
качество распознавания было заметно ниже, чем у более крупных Whisper-моделей
и специализированных моделей (GigaAM для русского, Parakeet TDT для
мультиязычности). Хранить несколько крупных моделей (150 МБ — 650 МБ каждая)
в бинарнике приложения — не вариант: размер установки взорвался бы для всех
пользователей независимо от того, какая модель им нужна.

## Решение

1. Встроенной модели больше нет. Первый запуск (`AsrFirstRunGate`,
   `main.dart`) проверяет наличие хоть одной установленной модели и, если её
   нет, предлагает выбрать и скачать одну из каталога — на Windows и Android
   одинаково.
2. Каталог (`lib/application/asr_model_catalog.dart`) — статический список из
   4 моделей (Whisper base/small, GigaAM v3, Parakeet TDT v3), каждая с
   оценкой 1–3 звезды по трём осям (русский / иностранный / скорость) и
   размером в МБ — видимыми в UI без клика.
3. Загрузка (`AsrModelManager.downloadAndInstall`) качает `potok-model.json`
   манифест по URL (тот же формат, что и ручная установка), затем каждый
   перечисленный файл рядом с манифестом, проверяет SHA-256 через
   существующий `_installFiles` — код установки не различает «скачано» и
   «выбрано вручную из папки».
4. Allowlist хостов (`github.com`, `*.githubusercontent.com`) — единственное
   сетевое использование во всём приложении (ADR-007). Редиректы проверяются
   на каждом хопе до перехода.
5. Модели хранятся в `getApplicationSupportDirectory()/models` — приватный
   для приложения каталог, который ОС удаляет автоматически при удалении
   приложения (Android: `getFilesDir()`-эквивалент; не публичное хранилище).
6. `AsrModelManager.deleteModel` — ручное удаление конкретного пака из UI,
   сбрасывает активный выбор, если удаляли активную модель.
7. Файлы моделей размещены в GitHub Releases (не в git-дереве — все
   существенно больше 100 МБ, git-лимита блоба). Один тег релиза на модель
   (`whisper-base`, `whisper-small`, `gigaam-v3`, `parakeet-tdt-v3`), чтобы
   избежать коллизий одноимённых файлов (`tokens.txt`, `encoder.int8.onnx`
   у разных моделей).

## Поддержанные архитектуры

Whisper (encoder+decoder+tokens) и NeMo-transducer (encoder+decoder+joiner+
tokens — GigaAM, Parakeet) через общий isolate-воркер
(`sherpa_asr_worker.dart`); выбор конкретного recognizer — по `model_type` в
манифесте (`sherpa_recognizer_factory.dart`), включая live-preview во время
записи.

## Последствия

- Установочный пакет уменьшился (не тянет модель по умолчанию); первая
  расшифровка недоступна до явного выбора пользователя.
- GigaAM v3 — только русский; для английского нужна вторая модель
  (Whisper) или Parakeet TDT v3 (мультиязычный, крупнее).
- Загрузка требует сети один раз (сама модель); распознавание речи
  по-прежнему работает полностью офлайн после установки.
