# ADR-014: live-fallback и прогресс локального ASR

Статус: принято 2026-07-22.

## Контекст

NeMo-transducer/Parakeet может вернуть пустую финальную гипотезу для записи с
десятками секунд начальной тишины, хотя короткие live-фрагменты уже содержали
распознанную фразу. Ранее live-текст использовался только для классификации и
терялся при создании durable-ревизии. Кроме того, sherpa-onnx выполняет offline
decode одним блокирующим нативным вызовом и не сообщает покадровый прогресс.

## Решение

- Для финального NeMo-transducer decode подрезается только краевая тишина
  длительнее двух секунд; сохраняется 400 мс контекста. Внутренние и короткие
  паузы не меняются. Whisper и live-фрагменты не затрагиваются.
- Уже показанный live-текст записывается в queued transcript revision как
  fallback. Непустой финальный результат заменяет fallback, пустой — сохраняет
  его. Retry переносит fallback в новую ревизию.
- Live UI показывает накопление очередного фрагмента и неопределённый прогресс
  во время decode.
- Для durable decode показывается явно подписанная оценка по длительности
  записи. Она ограничена 95% до фактического состояния `ready`, поэтому UI не
  заявляет ложное завершение. Точный процент появится только если runtime
  предоставит callback прогресса.

## Последствия

Текст, который пользователь уже видел в live-preview, не исчезает из-за
пустого финального ответа модели. Исходное аудио не изменяется и остаётся
доступным для повторной расшифровки. Все данные и вычисления остаются локальными.
