# ADR-018: нативная запись из Android-виджета

Статус: принято, 2026-07-23.

## Контекст

Пользователь должен одним нажатием на отдельный Android-виджет начать голосовую
заметку, не открывая `MainActivity`. Текущий recorder принадлежит Flutter UI и
не может работать, когда Flutter engine не запущен. Прямая запись Android-кода
в SQLite нарушила бы границы адаптеров и протокол staging → validate → publish.

Android 12+ ограничивает запуск foreground service из фона, а Android 14+
дополнительно проверяет while-in-use разрешение микрофона. Явное взаимодействие
с app widget является документированным исключением. Исключение действует
только при уже выданном `RECORD_AUDIO`; виджет не может показать системный
permission dialog самостоятельно.

## Решение

- Добавить отдельный однокнопочный `RemoteViews`-виджет. При выданном разрешении
  его кнопка отправляет явный `PendingIntent` в foreground service типа
  `microphone`; повторное нажатие останавливает запись.
- Foreground service пишет mono PCM16 WAV 16 kHz через Android `AudioRecord` в
  приватный `files/widget_recording_inbox`. Уведомление постоянно показывает,
  что микрофон активен, и содержит действие «Остановить».
- Inbox публикуется атомарной парой `UUID.wav` + `UUID.json`. Частичный WAV
  имеет суффикс `.partial`; после прерывания процесса его заголовок
  восстанавливается перед следующей обработкой.
- Нативный код никогда не открывает и не изменяет SQLite. При доступности
  Flutter application-координатор валидирует bounded inbox, создаёт staged
  audio note через `NotesService`, копирует WAV в `MediaStore`, вызывает
  `finishAudioNote` и только после успешного commit удаляет inbox-пару.
- После публикации используется обычная durable ASR-очередь, если локальная
  модель активна. Сетевого распознавания и скрытого fallback нет.
- Если `RECORD_AUDIO` не выдан, кнопка честно показывает необходимость
  разрешения и открывает существующий audio-capture flow для явного запроса.
- Одновременно допускается только одна запись. Виджет не начинает второй
  recorder, пока активна запись из Flutter UI, и наоборот.

## Последствия

- Нажатие начинает запись без визуального открытия приложения, в том числе при
  погашенном ранее экране приложения; Android всё равно показывает обязательный
  foreground notification и системный индикатор микрофона.
- Готовый файл переживает смерть Flutter-процесса. Между остановкой записи и
  следующим запуском Flutter заметка может находиться только в durable inbox и
  ещё не отображаться в списках/виджетах.
- Формат WAV увеличивает расход места примерно на 1.92 MB/мин при 16 kHz mono
  PCM16, зато не требует codec-плагина и непосредственно подходит локальному
  ASR. Длительность ограничена 30 минутами, сохраняется резерв свободного места.
- Решение Android-specific; Windows/domain semantics заметки не меняются.
