# ADR-005: Аудио — AAC/M4A baseline, пакет record, плеер just_audio

Статус: принят, дополнен WP-09 (2026-07-18)

## Решение
- Кодек записи: **AAC-LC в M4A**, mono, 44.1 kHz. Профили 48/64/96 kbps (~21/28/42 MB/час), default 64 kbps. Opus — только после compatibility-spike (FR-AUD-001/014).
- Запись: пакет `record` (Android MediaRecorder/AudioRecord, Windows MediaFoundation). Пишем в приватный staging, финализация — по протоколу ADR-004.
- Плеер: `just_audio` + `audio_session`; Windows backend — `just_audio_windows` (WinRT MediaPlayer). Скорости 0.75–2x, ±10 сек (FR-AUD-004/011).
- Для ASR: декод M4A → mono PCM 16 kHz через FFI-декодер плагина/платформы; исходник не модифицируется.
- Одна активная запись микрофона на устройство; длительная запись на Android — foreground service с уведомлением.
- Лимит записи по умолчанию 30 мин (настройка). Несколько аудио-вложений в заметке допустимы.
- Перед publish проверяются container signature, size и SHA-256. Recovery metadata фиксируется в hidden DB state до atomic rename.
- Удаление аудио — soft delete; physical purge удаляет байты/метаданные, но оставляет asset tombstone.

## Riski
- Windows-путь записи в `record` менее обкатан, чем Android — проверяется в вертикальном срезе WP-00; fallback — платформенный канал на MediaFoundation.

## Дополнение WP-09

- AAC/M4A остаётся компактным baseline при отсутствии активной ASR-модели.
- При активной локальной модели recorder пишет ASR-ready WAV PCM16 mono 16 kHz.
  Это временно устраняет отсутствующий M4A→PCM adapter одинаково на Android и
  Windows; цена явно показывается как ~110 MiB/ч.
- Выбор Windows input device выполняется через `record.listInputDevices()` и
  `RecordConfig.device`; device ID хранится в `app_meta`. Исчезнувший выбранный
  input не заменяется молча системным default.
