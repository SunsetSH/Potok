# ADR-015: системная тема и честный индикатор ASR

Статус: принято 2026-07-23.

## Контекст

Sherpa-onnx выполняет offline decode одним нативным вызовом и не отдаёт
покадровый прогресс. Расчёт времени по длительности записи оказался сильно
завышенным на быстрых устройствах. Также фиксированная тема приложения не
позволяла автоматически следовать системной светлой/тёмной теме, а Android
RemoteViews использовали одну статическую палитру.

## Решение

- Удалить расчёт оставшегося времени. Для `queued`/`recognizing` показывать
  только честный неопределённый индикатор и текст состояния.
- Индикаторы final и live ASR включены по умолчанию и отключаются одной
  локальной настройкой в `app_meta`; состояние job при этом всегда видно
  текстом, распознавание не меняется.
- Поддержать два режима темы: фиксированная тема либо `ThemeMode.system` с
  отдельно выбранными дневной и ночной темами из существующего набора.
- На Android и Windows системный режим использует `platformBrightness` —
  системную светлую/тёмную тему приложений. Windows Night Light является
  фильтром цветовой температуры и не считается сигналом смены темы.
- Flutter передаёт режим и три theme ID в ограниченный SharedPreferences-кэш
  Android-виджетов. RemoteViews самостоятельно выбирают дневную/ночную палитру
  по `Configuration.UI_MODE_NIGHT_*` и валидируют ID по allowlist. На Android
  12+ day/night-пары передаются через `RemoteViews.setColorInt`/
  `setColorStateList`, поэтому host переопределяет цвета при смене конфигурации
  без живого процесса приложения; на старых Android применяется палитра,
  актуальная на момент очередного обновления виджета.
- На Android 12+ внешние фоны виджетов используют системный радиус
  `system_app_widget_background_radius` и внутренний inset 1 dp. Цветная
  обводка поэтому остаётся внутри маски лаунчера и не обрезается на углах;
  внутренние карточки и кнопки сохраняют радиусы выбранной темы.

## Последствия

Интерфейс больше не показывает заведомо неточный countdown. Все домашние
виджеты используют фон, поверхности, акцент и текст активной темы. На Android
12+ смену системного day/night режима применяет widget host без запущенного
Flutter-процесса; на более старых версиях Android новая палитра появится при
очередном обновлении виджета. Настройки остаются локальными и не требуют новых
разрешений или сетевых вызовов.
